# 3.4 Analysis Methods

## 3.4.1 Data Preparation

Raw results are stored as JSONL files. Each line records one VLM response to one stimulus under one experiment variant. A corresponding manifest records the ground-truth metadata for each stimulus: edit type, degradation dimension, magnitude tier label, jitter-resolved numeric parameter values, layout properties, and image paths.

Results are joined with manifest metadata on `stimulus_id` at load time, producing a long-format table with one row per (stimulus, model, experiment) triplet and columns for all score dimensions, the degradation parameters, and the layout properties. All downstream analyses operate on this single table; cross-model results are produced by grouped operations on `(model, dimension, tier)` rather than by separate per-model passes.

Control stimuli (Section 3.4.5) are separated from degraded stimuli at load time and analysed separately. Sensitivity and threshold analyses run on degraded stimuli only.

Parse failures are excluded from all analyses. Parse success rate is reported per model and per experiment as a data quality indicator.

Each stimulus is assigned a continuous numeric magnitude derived from its degradation parameters. The conversion is dimension-specific:

| Dimension | Numeric magnitude |
|-----------|------------------|
| `color_offset` | ΔE (CIEDE2000) |
| `position_offset` | Euclidean distance in pixels: √(x² + y²) |
| `scale_error` | |scale\_error\_pct| (absolute percentage deviation) |
| `rotation` | |angle\_deg| (absolute rotation in degrees) |
| `font_weight_light` | 400 − weight (lighter direction; e.g. 400 → 300 → 200 → 100) |
| `font_weight_heavy` | weight − 400 (heavier direction; e.g. 400 → 500 → 600 → 700 → …) |
| `font_style` | Categorical severity: italic/normal = 1.0, oblique = 0.5 |
| `font_family` | Category distance: same-class fonts = 0.5, cross-class = 1.0 |
| `letter_spacing` | |letter\_spacing\_px| |
| `opacity` | 1.0 − opacity (opacity reduction) |
| `gaussian_noise` | Noise σ |
| `jpeg_compression` | 100 − quality |
| `blur` | Blur radius |

For stimuli with jitter-resolved parameters (drawn from a `min_X`/`max_X` range), the resolved scalar from the metadata is used directly rather than the range midpoint.

## 3.4.2 Detection and Sensitivity Metrics

The **detection rate** for dimension $d$ and model $m$ is the proportion of degraded stimuli (magnitude > 0) for which the model reported `detected_difference: true` in Experiment 1:

$$\text{DetectionRate}(d, m) = \frac{1}{N_{d,m}} \sum_{i \in S_{d,m}} \mathbb{1}[\text{detected\_difference}_i = \text{true}]$$

A dimension is classified as a **blind spot** for a given model when its detection rate is not significantly higher than the model's adjusted false positive rate $\text{FPR}_m$ (Section 3.4.5). This baseline-relative criterion ties the blind-spot designation to each model's own response tendencies rather than to a fixed cutoff. The adjusted FPR excludes the `alignment_error` dimension, which produces anomalously high raw FPR due to spatial hallucination rather than a general detection bias; the resulting per-model baselines are near-zero (≤ 3.2%), making the criterion practically very close to testing against chance while remaining methodologically consistent.

Because $\text{FPR}_m$ is estimated from a control set of comparable size ($n_{\text{ctrl}} \approx 31$ after excluding `alignment_error`), it is treated as a random quantity rather than a known constant. Significance is assessed with a one-sided binomial test ($H_0\colon p \leq \text{FPR}_m$) and 95% confidence intervals on the sensitivity margin $\Delta(d, m) = \text{DetectionRate}(d, m) - \text{FPR}_m$ are computed using Newcombe's hybrid score interval for the difference of two independent proportions. Benjamini–Hochberg correction is applied at $\alpha = 0.05$ across all (dimension, model) pairs.

To convey both effect size and statistical significance in a single view, results are presented as a **forest plot**. Each row corresponds to a degradation dimension; within a row, one point per model marks $\Delta(d, m)$, with horizontal error bars giving the 95% Newcombe CI. A reference line at $\Delta = 0$ marks the blind-spot boundary: points whose CI excludes zero (after BH correction) are sensitive on that dimension, points whose CI overlaps zero are blind. Dimensions are ordered by mean $\Delta$ across models.

The **score gap** between experiments for dimension $d$ and model $m$ is the mean difference between Experiment 2 overall_quality and Experiment 1 similarity_score on the same stimuli:

$$\text{Gap}(d, m) = \overline{\text{oq}_{E2}}_{d,m} - \overline{\text{similarity}_{E1}}_{d,m}$$

A positive gap indicates that Experiment 2 scores are inflated relative to perceptual similarity; a negative gap indicates the opposite. The pattern of gaps across dimensions tests whether the cost of instruction-level reasoning is dimension-general or dimension-specific.

## 3.4.3 Magnitude Tier Analysis

Each degradation dimension is sampled at multiple magnitude tiers (micro, tiny, small, medium, large, etc.), defined dimension-relatively so that "small" in `color_offset` corresponds to a different ΔE than "small" in `letter_spacing` does in pixels. Tier-level analysis allows direct cross-dimension comparison of the magnitude regime at which the response transitions from insensitive to sensitive.

For each (dimension, model) cell, mean detection rate (Experiment 1) and mean overall_quality (Experiment 2) are plotted against magnitude tier as psychometric curves, with one line per model on the same axes. Bootstrap 95% confidence intervals are shown as shaded bands.

The **threshold tier** for a (dimension, model) cell is the smallest magnitude tier at which detection rate reaches 0.5 (and separately 0.75). For dimensions where no tier reaches the threshold, the entry is "not crossed." This is a discrete analogue of a just-noticeable-difference estimate; it does not require interpolation and is stable under small per-tier sample sizes.

The threshold is reported in two forms: the tier label (e.g. "small") and the corresponding mean numeric magnitude in dimension-native physical units (e.g. ΔE for `color_offset`, pixels for `position_offset`, degrees for `rotation`). The physical value is pulled directly from the manifest's jitter-resolved parameters and enables cross-study comparison: a reader can compare, for instance, the ΔE at which a model's detection crosses 0.5 against perceptual-difference thresholds reported in prior work, without having to map through tier labels.


## 3.4.4 Statistical Tests

The primary statistical test for sensitivity is the **Spearman rank correlation** between numeric magnitude and the model's score (similarity_score for Experiment 1, overall_quality for Experiment 2):

$$\rho_s(\text{magnitude}, \text{score})$$

Spearman is used because scores are ordinal on a 1–5 scale and the magnitude distribution within a dimension is not guaranteed to be normal. A significant negative correlation (after FDR correction) indicates the model is sensitive to that dimension; a non-significant or near-zero correlation indicates a blind spot.

Correlations are computed for approximately 12 dimensions × 2–3 models × 2 experiments ≈ 60 primary tests. P-values are corrected using the Benjamini-Hochberg procedure at a false discovery rate of 5%. Both raw and corrected p-values are reported, with corrected values used for all significance claims.

A Spearman correlation matrix among the five Experiment 2 score columns (instruction_following, text_accuracy, visual_consistency, layout_preservation, overall_quality) is reported per model to characterise rubric internal structure. Strong intercorrelations indicate the model is collapsing the rubric into a single holistic judgment despite per-dimension instructions; weak intercorrelations indicate the dimensions are scored with some independence.

## 3.4.5 Control Stimulus Analysis

We use three control conditions to characterise model behaviour independent of the degradation manipulation. Each targets a distinct property of the judge: its baseline tendency to hallucinate differences, its sensitivity floor, and whether the rubric prompt actually probes instruction adherence. Each control set contains 34 stimuli, comparable in size to the typical per-dimension stimulus count of approximately 58 in the main experiments.

**Identity control (Experiment 1).** The model is shown an identical image pair: the correctly edited image presented as both reference and comparison. Any `detected_difference: true` response on these stimuli is a hallucination, and the **false positive rate** is estimated as

$$\text{FPR}(m) = \frac{1}{N_{\text{ctrl},m}} \sum_{i \in S_{\text{ctrl},m}} \mathbb{1}[\text{detected\_difference}_i = \text{true}]$$

Ideally near zero; elevated values indicate the model reports spurious differences in identical pairs. Results (Section 4.2.1) show that the raw FPR is dominated by a single anomalous dimension (`alignment_error`), where models hallucinate a difference because the relocated text looks spatially unusual relative to their internal priors rather than genuinely comparing the two images. Excluding this dimension, the adjusted FPR is ≤ 3.2% for all three models. This adjusted rate is used as the per-model null baseline in the blind-spot classification (Section 3.4.2).

**Source-target control (Experiment 1).** The model is shown the original unedited image as the reference and the correctly edited image as the comparison, a fully-applied edit with no degradation. The **detection rate** on these stimuli measures the model's sensitivity floor: whether it can detect a clean, ground-truth edit at all. Low detection here would indicate that null per-dimension detection rates cannot be unambiguously attributed to degradation invisibility, since the model is failing to register edits even when correctly applied. This control bounds the interpretation of the main experiment from below in the same way the identity control bounds it from above.

**Instruction-not-followed control (Experiment 2).** The model is shown the original unedited image as the "edited" result, along with the edit instruction. Since no edit was applied, the expected score across all rubric dimensions is 1. We report the **above-1 rate** (proportion receiving any score > 1) and the **high-rating rate** (proportion receiving any score ≥ 3) as a prompt-validity check on whether the rubric meaningfully measures instruction adherence.

Control metrics are reported per model alongside parse success rates and the score gap (Section 3.4.2) in a single failure-mode table.

## 3.4.6 Response Distribution Analysis

For each (dimension, model, experiment) cell, the histogram of scores given by the model is reported as a multi-panel figure. The distributions distinguish failure modes that mean-score and correlation summaries cannot: bimodal collapse to extremes, uniform mid-range scoring, and ceiling effects.

## 3.4.7 Model Agreement on Blind Spots
Each (dimension, model) cell is classified as blind or sensitive using the criterion in Section 3.4.2. The resulting blind/sensitive table indexed by dimension × model distinguishes dimensions where all models are blind (likely reflecting limitations common to current VLMs) from dimensions where models diverge (architecture-specific differences in perceptual or reasoning capabilities).

To summarise the degree of inter-model agreement with a single number, Fleiss' kappa is computed on the complete dimension × model binary classification table, treating each degradation dimension as a subject and each model as a rater. Rows where any model's classification is "unknown" (insufficient data) are excluded before computing kappa. A kappa near 1 indicates that models agree consistently on which dimensions are blind spots; a kappa near 0 indicates near-chance agreement; a negative kappa indicates systematic disagreement. This directly answers RQ1a's question of whether different VLMs have systematically different blind spots — high kappa means they share the same ones.
